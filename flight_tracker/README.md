# Flight Tracker

A real-time flight tracking app built with Flutter. It fetches **live aircraft
data directly from the device/browser** — there is no backend server, no
database, and no API key required to run it.

## How it works

- **Data source:** [OpenSky Network](https://opensky-network.org/) public
  REST API (`/api/states/all`). It's free, requires no signup for light use,
  and returns live aircraft position, altitude, speed, heading, and more.
- **No backend:** the Flutter app calls the OpenSky API straight from
  `FlightService` using the `http` package. There is nothing to host or
  deploy server-side.
- **Maps:** rendered with `flutter_map` using OpenStreetMap raster tiles,
  which also require no API key.

## Project structure

```
lib/
├── main.dart                     # App entry point, theme + provider wiring
├── models/
│   └── flight_state.dart         # Parses one OpenSky state vector
├── services/
│   └── flight_service.dart       # All network calls to OpenSky live here
├── providers/
│   └── flight_provider.dart      # App state: flights, region, refresh timer
├── theme/
│   └── app_theme.dart            # Light/dark Material 3 theme
├── screens/
│   ├── home_screen.dart          # Search, region picker, List/Map tabs
│   └── flight_detail_screen.dart # Full detail view for one aircraft
├── widgets/
│   ├── flight_list_item.dart
│   ├── flight_map.dart
│   ├── region_selector.dart
│   ├── search_field.dart
│   └── status_banner.dart
└── utils/
    ├── constants.dart            # API endpoint, regions, refresh interval
    └── formatters.dart           # Altitude/speed/heading/time formatting
```

Each layer only talks to the one below it (screens → widgets → provider →
service → model), so you can swap the data source, add caching, or add a
new screen without touching unrelated files.

## Setup

This repo ships only `lib/`, `pubspec.yaml`, and config files — not the
platform folders (`android/`, `ios/`, `web/`, etc.), since those are
generated by the Flutter tool itself. To get a runnable project:

1. Install the [Flutter SDK](https://docs.flutter.dev/get-started/install)
   if you don't already have it, and confirm it's healthy:
   ```bash
   flutter doctor
   ```
2. Unzip this project, `cd` into it, then generate the platform folders:
   ```bash
   flutter create . --project-name flight_tracker --org com.yourcompany
   ```
   This adds `android/`, `ios/`, `web/`, `macos/`, `windows/`, `linux/`
   alongside the existing `lib/` and `pubspec.yaml` — it will **not**
   overwrite the files in this repo.
3. Install dependencies:
   ```bash
   flutter pub get
   ```
4. Run it:
   ```bash
   flutter run                 # picks a connected device/simulator
   flutter run -d chrome       # run in a browser
   ```

## Building for deployment

```bash
# Android — installable APK
flutter build apk --release

# Android — Play Store bundle
flutter build appbundle --release

# iOS (must be run on macOS with Xcode installed)
flutter build ios --release

# Web — static site, deployable to any static host (Firebase Hosting,
# Netlify, Vercel, GitHub Pages, S3, etc.)
flutter build web --release
```

Build output locations:
- Android APK: `build/app/outputs/flutter-apk/app-release.apk`
- Android bundle: `build/app/outputs/bundle/release/app-release.aab`
- Web: `build/web/` (deploy this folder as-is)
- iOS: open `build/ios/` in Xcode to archive and submit, or use
  `flutter build ipa --release`

## Notes on the live data

- **Rate limits:** anonymous OpenSky access allows roughly one request
  every 10 seconds and ~400 requests/day per IP. The app refreshes every
  15 seconds by default (`AppConstants.defaultRefreshInterval`) and shows
  a friendly message if you're rate-limited. For heavier use, create a
  free account at opensky-network.org and pass credentials into
  `FlightService(username: ..., password: ...)` in `main.dart` — this
  raises the limit without adding any backend.
- **Regions:** the app queries a bounding box (continent) rather than the
  whole world by default, since "world" can return 10,000+ aircraft. Change
  the default or add new regions in `lib/utils/constants.dart`.
- **Web/CORS:** OpenSky's API generally allows cross-origin browser
  requests, but if you ever see CORS errors on `flutter build web`, that's
  a browser-side restriction from the API itself, not something fixable
  from within this codebase without introducing a proxy. Mobile and
  desktop builds are unaffected since CORS is a browser-only concept.

## App icon & name (optional)

Before publishing, you may want to set a custom app icon and name:
- App icon: add the
  [`flutter_launcher_icons`](https://pub.dev/packages/flutter_launcher_icons)
  dev dependency and point it at your artwork.
- App name: edit `android/app/src/main/AndroidManifest.xml`
  (`android:label`) and `ios/Runner/Info.plist` (`CFBundleDisplayName`)
  after running `flutter create .`.
